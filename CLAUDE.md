# 顧客対応ナビ — Claude Code 向けコンテキスト

社内スタッフがスマホで使う顧客対応ガイド。`customer-support-navi.html` の1ファイル完結（ビルド不要・依存なし）。出力は日本語。

## このツールの役割
問い合わせを3種別に分類し、確認すべきこと・考慮すべき観点を上から順にたどらせ、**不具合・要望は最後に起票文へ整形**してコピーできるようにする。元になった判断フローは「顧客対応の進め方」フローチャート。

## 守るべき制約（変更時に壊さない）
- **配色は6色のみ**。新しい色相を足さない。濃淡が必要なら下記の不透明度違いで出す。
  - `#424242`（文字・ダーク面・要望カテゴリのアクセント）
  - `#FFFFFF`（カード面）
  - `#EEEEEE`（境界線・チップ地）
  - `#FAFAFA`（アプリ背景）
  - `#BE9C59`（ゴールド＝基本アクセント／観点ドット／主要ボタン）
  - `#D15954`（赤＝不具合カテゴリのアクセントのみ）
  - 全色は `:root` のCSS変数で定義。直書きの新hexは追加しない。
- **モバイル前提**。縦1カラム、タップ領域44px以上、`prefers-reduced-motion` 尊重。
- **localStorage / sessionStorage は使わない**（claude.ai のArtifactプレビューで失敗する）。状態は `state` オブジェクト＝メモリ内のみ。永続化したい場合は別途相談。
- カテゴリ区別は色相を増やさず「ゴールド＝操作方法／赤＝不具合／チャコール＝要望」で行う。

## コンテンツの単一の真実（ここを編集すれば中身が変わる）
`FLOWS` 配列が全フローのデータ。各フローは `{id, c, cs, title, ex, motto, steps[], output}`。
- `steps[]`: `{title, items[], kanten?, noteLabel?, tip?}`
- `items[]`: `{q, sub?, kanten?}`
- `kanten`: `KANTEN` のキー配列（`fact` / `abstract` / `customer` / `gcd` / `oneToN`）
- `output.build(n)`: 起票文テンプレ。`n['s0']` 等が各ステップのメモ、`n['out_xxx']` が起票欄。
- レンダリングは `FLOWS` から自動生成。**ステップ・項目・観点・カテゴリの追加はデータを足すだけ**でUIに反映される。

## よくある編集と勘所
- 観点の文言変更 → `KANTEN`
- 起票テンプレの項目変更 → 該当フローの `output.build` と `output.fields`
- 新カテゴリ追加 → `FLOWS` に要素追加＋`IC` にアイコンSVG追加（既存6色のみ使用）
- 進捗・チェックはリロードで消える仕様（1件ごとに完結させる前提）

## 検証
ブラウザで `customer-support-navi.html` を直接開けば動作確認できる。スマホ幅（〜400px）で崩れないことを必ず確認。
